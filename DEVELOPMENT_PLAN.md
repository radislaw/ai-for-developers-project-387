# План развития Booking App

## Текущее состояние

Приложение для онлайн-записи к специалисту (аналог Calendly). Стек: React + Mantine UI + React Query (фронтенд), Express + in-memory store (бэкенд), TypeSpec API contract, Playwright E2E тесты, деплой на Render.

### Реализованный функционал
- Лендинг-страница
- Каталог услуг (event types)
- Флоу бронирования: выбор даты → выбор слота → форма → подтверждение
- Админ-панель: просмотр предстоящих бронирований, создание услуг
- Отображение информации о владельце

## Баги и проблемы

### Критические
1. **In-memory store** — все данные теряются при перезапуске сервера. Нет персистентности.
2. **Нет аутентификации** — админ-панель доступна любому пользователю по `/admin`.
3. **Конфликт CORS** — в `backend/src/index.ts:13` захардкожены только `localhost:5173` и `5174`. Продакшен-домен не указан, фронтенд на Render может не работать.

### Средние
4. ~~**Невалидный `id` при создании услуги**~~ — **ИСПРАВЛЕНО**: сервер генерирует `id` через `randomUUID()`, фронтенд отправляет `Omit<EventType, 'id'>`, мёртвый 409-чек удалён (#4).
5. **Нет валидации email на бэкенде** — `backend/src/routes/bookings.ts:20` проверяет только наличие поля, но не формат email.
6. **Нет обработки ошибок API на фронте** — при ошибке создания бронирования (`Book.tsx:57-70`) нет показа ошибки пользователю, только `onSuccess` навигация.
7. **Слоты не фильтруются по прошлому времени** — `backend/src/routes/slots.ts` возвращает слоты за весь день, включая уже прошедшие. Пользователь может выбрать прошедшее время.
8. **`handleDateSelect` сбрасывает step неверно** — `Book.tsx:46` устанавливает `step: 'select-date'` при выборе даты, но не сбрасывает `selectedSlot`, хотя `selectedSlot` сбрасывается. Если пользователь выбрал слот, затем другую дату — форма может отобразиться некорректно.

### Незначительные
9. **Нет `favicon`** — стандартная иконка Vite в продакшене.
10. **Хардкод owner** — `backend/src/store.ts` содержит статические данные владельца без возможности изменения.
11. **`Calendar.tsx` не показывает подсказку при пустых `slotsData`** — `hasAvailableSlots` возвращает `true` при пустом массиве, что вводит в заблуждение (все даты показывают зелёную точку).
12. **Нет `meta`-тегов** — нет SEO и Open Graph разметки.
13. **Админ не может удалять/отменять бронирования**.
14. **Админ не может редактировать/удалять услуги**.

## Новые фичи

### Приоритет 1 (MVP-улучшения)
1. **Персистентное хранилище** — заменить in-memory store на SQLite (better-sqlite3) или JSON-файл. Данные сохраняются между перезапусками.
2. **Отмена бронирования** — добавить endpoint `DELETE /bookings/:id`, кнопку отмены в админке и на странице подтверждения (по токену).
3. **Показ ошибок пользователю** — toast-уведомления при ошибках API (ошибка сети, конфликт слота, валидация).
4. **Фильтрация прошедших слотов** — бэкенд не отдаёт слоты с `startAt < now`.
5. **Валидация email на бэкенде** — проверка формата перед созданием бронирования.

### Приоритет 2 (расширение функционала)
6. **Редактирование услуг** — `PUT /event-types/:id`, форма редактирования в админке.
7. **Удаление услуг** — `DELETE /event-types/:id` с подтверждением.
8. **Детальная страница бронирования** — показать все детали записи (услуга, время, имя, email, заметки) на отдельной странице.
9. **Фильтрация бронирований в админке** — по дате, по услуге, поиск по имени/email.
10. **Пагинация бронирований** — при большом количестве записей.
11. **Email-подтверждение** — отправка письма гостю после бронирования (Nodemailer + SMTP или Resend/SendGrid).
12. **Часовой пояс гостя** — показывать время в часовом поясе гостя, а не только владельца.

### Приоритет 3 (качество и UX)
13. **404 страница** — кастомная страница для несуществующих маршрутов.
14. **Error Boundary** — глобальная обработка ошибок React.
15. **Unit-тесты** — тесты для хуков и утилит (Vitest).
16. **Мобильная адаптация** — проверить и улучшить отображение на мобильных устройствах (особенно календарь и навбар).
17. **SEO** — meta-теги, Open Graph, title для каждой страницы.
18. **Favicon** — кастомная иконка приложения.
19. **Тёмная тема** — переключатель темы (Mantine поддерживает из коробки).
20. **Анимации переходов** — плавные переходы между шагами бронирования.

### Приоритет 4 (продвинутый функционал)
21. **Аутентификация админа** — JWT или session-based авторизация для админ-панели.
22. **Настройка рабочих часов** — владелец задаёт своё расписание (время начала/конца, обед, выходные).
23. **Настройка услуг** — привязка услуг к конкретным дням недели.
24. **Повторяющиеся бронирования** — возможность записи на регулярной основе.
25. **Календарная интеграция** — экспорт в Google Calendar / iCal.
26. **Мульти-владелец** — поддержка нескольких специалистов с各自的 расписанием.
27. **Webhook-уведомления** — уведомление внешних систем о новых бронированиях.
